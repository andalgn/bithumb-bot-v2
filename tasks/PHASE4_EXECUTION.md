## 작업 목록

### 4.1 execution/partial_exit.py (새로 작성)
- 전략 A: +3%→30%, +6%→30%, 나머지 트레일링
- 전략 B: BB 중간선→50%, 나머지 트레일링
- 전략 C: 트레일링 ATR×3.0 (부분청산 없음)
- 전략 D: 고정 TP 1.5% 전량 (부분청산 없음)
- 각 트렌치를 개별 주문 실행
- 수수료 누적 계산 반영
- journal에 트렌치별 기록 → 종료 시 합산

### 4.2 트레일링 스톱 구현
- Active: ATR × Tier별 배수 (T1:2.5, T2:2.0, T3:1.5)
- Core(승격): 1H ATR × 2.5, 단 Active 손절폭 2배 이내
- 활성화: +3% 도달 시
- 콜백률: 1%

### 4.3 체결 정책 구현
- 지정가 우선 → 30초 미체결 시 시장가 전환
- 슬리피지 허용: Tier별 ×2 (초과 시 취소)
- 재시도: 최대 3회, 500ms × 2^n
- 가격 이탈: 신호가 대비 ±0.3% 초과 시 포기
- 부분체결: 잔량 × 현재가 < 5,000원이면 취소

### 4.4 청산 우선순위 충돌 해결
- Core: 손절 > 강등 > 부분청산 > 트레일링
- Active: 손절 > 부분청산 > 타임아웃
- 한 캔들에서 다수 이벤트 → 보수적 규칙 우선

### 4.5 테스트
- 부분청산 3트렌치 정상 실행
- 수수료 누적 정확성
- 우선순위 충돌 시나리오
