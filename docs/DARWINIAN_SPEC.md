# Darwinian 자가 학습 명세

## 1. 개요
전략 자체가 아닌 **파라미터 세트**가 경쟁하는 경량 Darwinian.
LLM 없이 순수 연산, 추가 비용 0.

## 2. 구조

| 구성 | 설명 |
|------|------|
| Population | 파라미터 조합 **20~30개** 동시 유지 (미니PC 8코어 활용) |
| Champion | 실제 거래 실행하는 파라미터 세트 1개 |
| Shadow | 매 사이클마다 "진입했을까?" 기록하는 가상 포지션 |
| 3그룹 구성 | Shadow 1~7: ±10%(미세) / 8~15: ±20%(탐색) / 16~20+: 혁신적 |
| 주간 토너먼트 | 7일간 Shadow 성과 비교, 상위 5개 생존, 하위 변이 |
| 챔피언 교체 | **월 1회 이하**, Composite Score 기준 |
| 교체 쿨다운 | 교체 후 **14일간** 재교체 금지 |

## 3. 단계별 프로세스

| 단계 | 주기 | 조건 | 행동 |
|------|------|------|------|
| 관찰 | 매일 | Shadow 성과 기록, 체결비용 보정 | 랭킹만 갱신, 교체 없음 |
| 선별 | 매주 | 하위 Shadow 정리, 표본 미달 제외 | 후보군 3개 유지 |
| 교체 | 월 1회 이하 | Composite Score 우위 + 검증 통과 | 챔피언 1개 교체 가능 |

## 4. Composite Score (챔피언 교체 기준)

Sharpe 단독 비교 **금지**. 아래 복합 지표 가중합으로 판단.

| 지표 | 가중치 | 교체 최소 조건 |
|------|--------|--------------|
| Expectancy | 30% | 챔피언 대비 +10% 이상 |
| Profit Factor | 20% | 챔피언 대비 +0.15 이상 (PF = 총 이익 합 / |총 손실 합|) |
| MDD | 20% | 챔피언보다 나빠도 +20% 이내 |
| Sharpe | 20% | 챔피언 대비 +10% 이상 |
| Execution Quality | 10% | 실패율/슬리피지/부분체결 보정 후 우위 |

## 5. 교체 검증 조건

- 전략별 최소 거래 수 충족 (TRADE_SCHEMA.md 참조)
- 국면별 최소 10건 충족
- **3개 윈도우 모두 통과**: 최근 14일 + 직전 14일 + 최근 30일에서 챔피언 대비 열세가 아닐 것
- 교체 후 14일 쿨다운
- 동시에 2개 이상 파라미터 군 변경 금지

## 6. Shadow vs Live 비교 보정

Shadow는 가상이므로 공정한 비교를 위해 보정 적용:

| 항목 | Shadow 처리 | 보정 |
|------|-----------|------|
| 체결가 | 신호 시점 종가 | 해당 코인의 실제 Tier에 따른 슬리피지 차감 (T1:0.05%, T2:0.1%, T3:0.2%) |
| 수수료 | 미반영 | 매수+매도 0.5% 차감 |
| 부분체결 | 항상 전량 체결 | 보정 없음 (보수적 계산) |
| 주문 실패 | 발생 안 함 | 실제 실패율 기반 페널티 |
| 비교 기준 | — | Shadow net_pnl × 0.85 vs Live net_pnl |

## 7. 돌연변이 범위 제한
| 파라미터 | 1회 변이폭 |
|---------|-----------|
| RSI 범위 | ±3 이내 |
| ATR 배수 | ±0.3 이내 |
| 신뢰도 임계값 | ±5% 이내 |
| TP/SL 비율 | ±0.5% 이내 |

## 8. 과적합 방지
- In-sample(14일) Sharpe > 1.0 AND Out-of-sample(14일) Sharpe > 0.7
- 차이 50% 이상이면 과적합 → 해당 변이체 폐기
- 레짐 전환 후 48시간은 챔피언 교체 금지

## 9. 활성화 시점
- **Shadow 추적**: Phase 1부터 데이터 축적 시작 (스텁 기록)
- **Shadow 20~30개 본격**: Phase 5에서 활성화
- **주간 토너먼트**: Phase 5에서 활성화
- **Walk-Forward / Monte Carlo / 민감도**: Phase 5에서 활성화
- **챔피언 교체**: LIVE 전환 후 최소 30일 데이터 축적 후 활성화

## 10. 백테스트 검증 연동
Darwinian 교체 판정 시 아래 검증 결과도 함께 확인:
- Walk-Forward 4구간 중 3개 이상 통과 필수
- Monte Carlo 하위 5% PnL > -2% 필수
- 민감도 CV < 0.3인 파라미터만 교체 대상
- 하나라도 미충족 시 교체 보류
