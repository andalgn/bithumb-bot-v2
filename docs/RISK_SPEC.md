# RiskGate 상태 전이 및 체결비용 모델

## 1. RiskGate 우선순위 체계

모든 주문은 반드시 RiskGate를 거쳐야 함. 복수 규칙 동시 트리거 시 상위가 오버라이드.

| 우선순위 | 규칙 | 유형 | 효과 |
|---------|------|------|------|
| P0 | CRISIS 전량 청산 | Hard Stop | 모든 포지션 즉시 청산 + 전략 정지 |
| P1 | 인증 오류 (401/403) | Hard Stop | 전체 격리 600초 + 관리자 알림 |
| P2 | 총 DD 20% 초과 | Hard Stop | 신규 BUY 전면 차단 + 관리자 수동 승인 전까지 |
| P3 | 월간 DD 12% 초과 | Soft Stop | 신규 BUY 차단, 기존 포지션 유지 |
| P4 | 주간 DD 8% 초과 | Soft Stop | 신규 BUY 차단 |
| P5 | 일일 DD 4% 초과 | Soft Stop | 신규 BUY 차단, 00:00 KST 리셋 |
| P6 | 총 익스포저 90% 초과 | Soft Stop | 신규 진입 차단 |
| P7 | 전역 격리 (8회 실패) | Soft Stop | 60초 전체 격리 |
| P8 | 종목 격리 (3회 실패) | Soft Stop | 120초 해당 종목 격리 |
| P9 | 연속 손실 5회 | Soft Stop | 진입 중단, 일일 리뷰에서 평가 |
| P10 | 쿨다운 (60분) | Filter | 해당 종목 재진입 금지 |

## 1-1. DD 기준값(Base) 관리 방식

| 기간 | Base 설정 시점 | 갱신 방식 |
|------|-------------|----------|
| 일일 (daily) | 매일 00:00 KST 리셋 시 | 기간 리셋 시에만 현재 equity로 재설정 |
| 주간 (weekly) | 매주 월요일 00:00 KST | 기간 리셋 시에만 현재 equity로 재설정 |
| 월간 (monthly) | 매월 1일 00:00 KST | 기간 리셋 시에만 현재 equity로 재설정 |
| 총 (total) | 최초 시작 시 | HWM(High Water Mark) 방식 — 매 사이클마다 상향만 가능 |

> **주의**: daily/weekly/monthly base는 기간 내 HWM으로 매 사이클 상향하지 않음. 해당 기간 시작 시점의 equity가 기준.

## 2. Hard Stop vs Soft Stop

| 구분 | Hard Stop | Soft Stop |
|------|-----------|-----------|
| 신규 진입 | 차단 | 차단 |
| 기존 포지션 | 즉시 청산 시도 | 유지 (정상 SL/TP로 관리) |
| 복구 방법 | 관리자 수동 승인 필요 | 조건 해제 시 자동 복구 |
| 텔레그램 알림 | 즉시 + 반복 (5분 간격) | 발생 시 1회 |
| 로그 레벨 | CRITICAL | WARNING |

## 3. 오류 복구 후 재개 기준

| 오류 유형 | 재개 조건 | 재개 후 제한 |
|----------|----------|-------------|
| 인증 오류 | API 재인증 성공 + 5분 대기 | 첫 1시간 Active Pool만 |
| 총 DD 20% | 관리자 텔레그램 승인 | 재개 후 7일간 risk_per_trade 50% 축소 |
| CRISIS | 6봉 연속 정상 확인 | Reserve Pool에서 먼저 소액 테스트 |
| 전역 격리 | 300초 비활성 후 자동 리셋 | 제한 없음 |

## 4. Expected Edge 필터

Layer 3에서 진입 직전 계산. 양수일 때만 진입 허용.

```python
expected_edge = expected_return - (fee + slippage + failure_penalty)

# expected_return: 해당 전략 최근 30일 Expectancy (표본 미달 시 0)
# fee: 0.50% (매수+매도)
# slippage: Tier별 × 2 (매수+매도) → T1:0.10%, T2:0.20%, T3:0.40%
# failure_penalty: 최근 30일 실패율 × 평균손실액

if expected_edge <= 0:
    reject("Expected Edge negative")
```

## 6. 체결 정책

| 항목 | 정책 | 비고 |
|------|------|------|
| 기본 주문 | 지정가 우선 | 수수료 절감 |
| 미체결 타임아웃 | 30초 | 1초 간격 폴링 |
| 타임아웃 후 | 시장가 전환 | 슬리피지 허용 범위 내 |
| 슬리피지 허용 | Tier별 추정 × 2 | 초과 시 주문 취소 |
| 부분체결 | 잔량 × 현재가 < 5,000원이면 취소 | 최소 주문금액 |
| 실패 재시도 | 최대 3회, 간격 500ms × 2^n | 지수 백오프 |
| 재시도 가격 이탈 | 신호가 대비 ±0.3% | 초과 시 주문 포기 |

## 7. 슬리피지 모델

백테스트 및 Shadow 비교 시 적용:

| Tier | 슬리피지 추정 | 근거 |
|------|-------------|------|
| Tier 1 (BTC, ETH) | 0.05% | 높은 유동성 |
| Tier 2 (XRP, SOL 등) | 0.10% | 중간 유동성 |

실제 거래 슬리피지를 지속 기록하여 분기별 보정.

## 8. 부분청산/트레일링 우선순위

Core 포지션: 손절 > 강등 > 부분청산 > 트레일링
Active 포지션: 손절 > 부분청산 > 타임아웃 청산
한 캔들에서 다수 이벤트 시 더 보수적인 규칙이 우선.

## 9. 체결 비용 모델 세부

### 가격 라운딩 방향
Normalizer의 가격 정규화 시 **유리한 방향으로 라운딩**:
- **매수(BUY)**: 가격 **내림**(floor) — 더 낮은 가격에 매수 시도
- **매도(SELL)**: 가격 **올림**(ceil) — 더 높은 가격에 매도 시도

### 부분청산 수수료 처리
- 부분청산 시 해당 비율의 수수료는 **즉시 반영** (net_pnl에 차감)
- 전체청산 시에는 **나머지 미청산 분의 수수료만** 추가 차감
- 이전에 부분청산에서 이미 반영된 수수료를 이중 차감하지 않음
