# PRD v1.3 전체 요약

## 왜 개편하는가
1. 코인별 변동성 차이 미반영 → 거짓 신호 빈발
2. 진입 조건 과도한 AND → 진입 못함, 유휴자금 90%+
3. 15M 단일 TF → 상승 추세에서도 조기 청산, 수익 잘림
4. P0 버그 5개 (capital=0, Kelly 오염, Risk Engine 미연결 등)
5. God Object 구조 (main_rule.py 920줄)

## 무엇을 만드는가

### 전략 체계
- **3단계 파이프라인**: 환경 필터 → 신호 트리거 → 확인 필터
- **5단계 국면**: STRONG_UP / WEAK_UP / RANGE / WEAK_DOWN / CRISIS + 보조 플래그
- **5개 전략**: 추세추종(A) / 반전포착(B) / 브레이크아웃(C) / 스캘핑(D) / DCA(E)
- **3그룹 점수 컷오프**: A/B ≥72, C/D ≥78, E ≥68 (Full) | Probe는 각각 55/62/53 이상

### 자금 구조
- **3풀**: Core(60%) + Active(30%) + Reserve(10%)
- **Pool 기반 사이징**: Active 잔액×3%, Core 잔액×10% (총자산 기준 아님)
- **승격 시스템**: Active에서 진입 → 조건 충족 시 Core로 승격 (15M 진입, 1H 청산 분리)

### 안전장치
- **RiskGate**: P0~P10 우선순위 체계, Hard Stop / Soft Stop 구분
- **Expected Edge 필터**: 예상순엣지 ≤ 0이면 진입 금지
- **호가 잔량 필터**: Tier별 차등 (T1:5배, T2:3배, T3:2배)

### 자가 학습
- **일일 리뷰**: 규칙 기반 (LLM 없음), 매일 00:00 KST
- **주간 심층 분석**: DeepSeek-chat 1회 호출, 메타 수준 패턴 분석
- **경량 Darwinian**: Shadow 추적 → 월 1회 Composite Score 기반 챔피언 교체

### LIVE 승인
- **28일 PAPER** + 120건 거래 + 전략별 Expectancy>0 + MDD<8% + 8항목 체크리스트

## 핵심 설계 원칙
1. **구조에서 견고성을** — 파라미터 최적화가 아닌 MTF 필터링 + 적응형 청산
2. **승률보다 기대값을** — 전략별 다른 목표 (추세: R:R 2.5:1, 반전: 승률 60%)
3. **LLM은 리뷰에만** — 실시간 판단은 룰 기반 (백테스트 가능), LLM은 주간 메타 분석
4. **사이징은 금액, RiskGate는 허가** — 역할 분리
5. **과적합 방지** — In/Out-of-sample 이중 검증, Darwinian 교체 월 1회 + 14일 쿨다운
