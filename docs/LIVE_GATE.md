# LIVE 승인 게이트

## 1. 자동 검증 조건

아래 **모든** 요건 충족 전까지 LIVE 전환 금지.

| 영역 | 최소 기준 | 판정 |
|------|----------|------|
| PAPER 기간 | 최소 28일 연속 | 실제 경과 시간 기반 (첫 거래~현재) |
| 총 거래 수 | 120건 이상 | 전략별/국면별 표본 일부 확보 |
| 전략 유효성 | 활성 전략 모두 Expectancy > 0 | 전략별 실제 거래 데이터에서 개별 계산 |
| 손실 한도 | PAPER MDD < 8%, 일일 DD < 3.5% | max_daily_dd 추적값 사용 (순간 DD 아닌 기록된 최대값) |
| 운영 안정성 | 가동률 99%+, 인증오류 미해결 0건 | 실제 가동 시간 / 전체 경과 시간으로 계산 |
| 체결 품질 | 실슬리피지 모델 오차 ±25% 이내 | 비용 모델 검증 완료 |
| Walk-Forward | 최근 4구간 중 3개 이상 통과 | 과적합 아님 확인 |
| Monte Carlo | 하위 5% PnL > -2% | 운 의존 아님 확인 |
| 수동 점검 | 체크리스트 100% 완료 | 최종 승인 필요 |

## 2. 수동 체크리스트

| # | 항목 | 확인 |
|---|------|------|
| 1 | 모든 주문 경로가 RiskGate를 반드시 통과하는지 코드 리뷰 완료 | □ |
| 2 | 텔레그램 Hard Stop / Soft Stop 알림 수신 확인 | □ |
| 3 | 401/403 인증오류, 주문 실패, 미체결 재호가 시나리오 리허설 완료 | □ |
| 4 | 부분청산/트레일링/강등 동시 발생 시 우선순위 검증 완료 | □ |
| 5 | Shadow/Live 성과 테이블과 journal 스키마 일치 검증 완료 | □ |
| 6 | Pool 간 자금 이동(승격/강등) 정합성 검증 | □ |
| 7 | CRISIS 전량청산 → 복구 → 재개 전체 흐름 테스트 | □ |
| 8 | 일일 리뷰 + 주간 DeepSeek 분석 정상 작동 확인 | □ |

## 3. LIVE 전환 절차

1. 위 자동 검증 + 수동 체크리스트 100% 통과
2. 관리자(Cho) 텔레그램 `/resume` 명령으로 LIVE 전환
3. 전환 후 첫 7일: risk_per_trade 50% 축소 상태로 운영
4. 7일 후 정상 파라미터 복원

## 4. 미달 전략 처리

- PAPER 28일 후 Expectancy ≤ 0인 전략은 LIVE에서 자동 비활성화
- 억지로 활성화하지 않음
- 비활성 전략은 Shadow 모드로 계속 추적하여 데이터 축적
- 다음 월간 리뷰에서 재평가
